01
The manifesto

Why we built this.

Voice is the oldest interface. Typing is a translation we’ve been doing for so long we forgot it was a translation at all.

OpenFlow closes that gap — for English speakers, for Hindi speakers, for the millions of us who think in two languages at once and stop mid-sentence because the keyboard can’t keep up.

The good versions of this idea cost fifteen dollars a month. We don’t think the right to write at the speed of thought should sit behind a paywall.

So we built one. We’re giving it away. And we’re asking you to make it better.

— Oltaflock, 2026
02
Design principles

Four rules that shape every decision.

Every interface choice, every line of microcopy, every default value should be defensible against these four lines.

P 01

Quiet by default.

OpenFlow lives in your tray, not on your screen. It surfaces when summoned, disappears when done. No notifications, no nags, no toasts. The interface is your existing apps; we’re just the cursor.

P 02

Bilingual without apology.

Hindi is not a localization layer. It’s a first-class mode. Hinglish is not a bug to suppress. The dictionary system, the prompts, the type stack — all assume you may switch mid-sentence, and that’s fine.

P 03

Yours to inspect, yours to break.

Every model, prompt, hotkey, and config sits in a plain text file you can read. No telemetry, no analytics, no cloud sync we didn’t ask permission for. If you don’t like a decision, the code is right there.

P 04

Craft over cleverness.

A two-second latency budget. A button that depresses exactly four pixels. A waveform that breathes at 1.4 seconds. The small things compound — software that feels considered is software that gets used.

03
Identity · Logo system

A mark for the breath that becomes a word.

Three concentric arcs, asymmetrically rotated. They read as sound ripples, as the letter O, as the Devanagari ओ. At the center, a single terracotta dot — the moment air becomes intent.

Variants

Mark / on paper
Mark / on dark
Mark / mono on accent
✓ Do

Keep clear space equal to the diameter of the inner arc on all sides. Scale proportionally — never below 24px on screen, 8mm in print.

✗ Don't

Don’t recolor the arcs to anything but ink black or paper cream. Don’t rotate the mark. Don’t outline, drop-shadow, or place inside a container. Don’t change the dot from terracotta.

06
Voice & tone

How OpenFlow speaks.

Plain. Considered. Slightly literary when there’s room for it. Never cute, never corporate, never apologetic. Talks to the user like a colleague who shipped something good.

✓ Voice
"Listening. Release the key when you’re done."

Direct, calm, gives the user back agency. No exclamation marks, no “Great!”, no AI cheerleading.

✗ Not this
"🎙️ I’m ready to listen! Press your hotkey to start dictating!"

Emojis, exclamation points, performative friendliness. Talks at the user instead of working alongside them.

✓ Voice
"Microphone permission denied. Open System Settings → Privacy → Microphone."

States what happened, states the fix, gets out of the way. Treats the user as capable.

✗ Not this
"Oops! It looks like we don’t have permission to use your microphone yet. Let’s fix that together!"

“Oops”, “let’s”, “together” — false warmth that wastes the user’s time.

Interface · While recording

A pill, a pulse, a waveform.

When the hotkey is held, a small dark pill appears at the bottom-center of the active display. Heart-rate pulse on the dot. Live waveform reacts to actual audio amplitude. The mode label confirms what OpenFlow is going to do with what you say.

Five minutes from install to dictating.

Four steps: welcome, permissions, API key, mic test. Each step does one thing. Skip is always available. The progress bar is terracotta because progress should feel like motion, not duty.

Open source

A repository made to be forked.

OpenFlow ships as a public GitHub repository under the MIT license. No CLA, no contributor agreement, no telemetry. Every file in the repo is documented; every issue gets a human reply within a week. The bar for contribution is "does it make OpenFlow more useful or more honest".

$ git clone https://github.com/oltaflock/openflow
Cloning into 'openflow'… done.
$ cd openflow && ./install.sh
✓ Python 3.10+ detected ✓ ffmpeg installed ✓ Downloading Whisper medium (1.5 GB)… ✓ All dependencies installed → Launch OpenFlow to complete setup
# That's it. No account, no signup, no telemetry.

This document, committed to the repo as an HTML file. Renders on GitHub Pages. So forks can adopt the system or fork it.

docs/brand.html
License recommendation

MIT License — chosen because it’s the path of least friction for community adoption. Permits commercial use, modification, distribution, and private use. Forks can do whatever they want, including charge for their fork. We chose this consciously: the goal is for the idea of free voice dictation to spread, not for OpenFlow to win.

Community channels

Lightweight by design. No Discord guild of 50 unused channels. Just the four places where useful conversation actually happens for a tool this size.

GitHub Issues

Bugs, feature requests, language support requests. Triaged weekly. Labels: bug · enhancement · good-first-issue · language.

GitHub Discussions

Q&A, show-and-tell of custom dictionaries, requests for help. Pinned thread: “What did you dictate today?”

The newsletter

One email per release. New features, contributor shout-outs, the one or two issues we wish more people would look at. Hosted on Buttondown.

Mastodon / Bluesky

@openflow on both. Build-in-public updates, feature previews, accessibility notes. No X / Twitter — by choice.
